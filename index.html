Introdução

Este guia é uma leitura rápida para o trabalho em gerenciamento de informações dentro de um ambiente humanitário.

Muitas das dicas dadas podem ser óbvias, mas elas resolvem erros repetidos encontrado neste setor.

Um escritório temporário após o tufão Haiyan

Introdução

Dados bem gerenciados e precisos são essenciais para tomada de decisão dentro do setor humanitário. O gerenciamento de informações pode ajudar a lançar luz sobre estrangeiros ou mudar rapidamente ambientes que fornecem contexto aos tomadores de decisão. Fluxos de informações bons e bem gerenciados podem tornar as organizações mais responsiva e eficiente, levando a maior eficácia.

Como usar

As dicas aqui não devem ser considerado regras rígidas, mas sim de orientação geral. O contexto de uso pode exigir as vezes abordagens diferentes e únicas.

Cada página fornecerá uma dica e uma breve descrição. Se você quiser descobrir mais sobre uma dica específica, pressione no teclado ou deslize o dedo para baixo. Aqui você pode encontrar mais informações e links para recursos que fornecem mais materiais de leitura e aprendizagem.

O Básico

01

Use uma planilha para dados numéricos

Houve muitos exemplos no passado sobre respostas de dados humanitários numéricos sendo coletados em um documento de processamento de texto quando eles seriam mais adequados em uma planilha.

Isso significa agregação simples, operações e a análise não pode ser concluída sem primeiro importar para uma planilha. Tabelas de palavras também pode ser formatado de uma maneira que não traduz bem para a planilha, o que significa mais trabalho antes de poder ser usado para análise.

01

O Microsoft Excel é o mais usado, mas existem opções gratuitas para usar com um pouco menos de funcionalidades.

Google Spreadsheets: Webpage

02

Conecte-se à comunidade de gestores de informação trabalhando na área

Na maioria dos casos, onde trabalho humanitário está ocorrendo haverá outras organizações trabalhando em projetos. Participando de reuniões locais e ingressando em grupos relevantes do Skype, você pode criar contato com outros gestores de informações. Ao compartilhar conhecimento e colaborar em resolver problemas de trabalho isso pode ser concluído rapidamente e eficientemente.

03

Salvar frequentemente

Esta é uma lição aprendida rapidamente pela maioria através da experiência prática. Computadores pode ser coisas temperamentais, coisas podem acontecer enquanto você assiste seu computador travar e perder horas de trabalho. Salve-se da angústia e salve com frequência!

04

Salve várias versões

Você pode se arrepender de uma mudança que você fez no seu trabalho. É melhor salvar várias versões à medida que avança para que você possa reverter facilmente todas as alterações feitas.

05

Use nomes de arquivos compreensíveis

Existem inúmeros atributos que você pode incluir em um nome de arquivo. O aspecto mais importante é garantir que seja compreensível para si mesmo e também para outras pessoas que possam usar seu arquivo sem o contexto completo. Alguns componentes sugeridos:

  • Data - YYYYMMDD
  • Descrição
  • Suas Iniciais
  • Número de versão
06

BackUp de seu dado

Dados salvos pode às vezes se perder. Isso pode ser devido a um vírus, problema no disco rígido, erro do usuário ou dezenas de outras razões.

Com o crescimento de serviços na nuvem para salvar arquivos, o trabalho pode ser salvo online gratuitamente.

Dropbox usado na resposta ao terremoto da IFRC no Nepal em 2015

06
Existem alguns serviços em nuvem que fornecem contas gratuitas para uma pequena quantidade de hospedagem. Eles podem ser usados para sincronizar uma pasta no seu computador e ser armazenada remotamente. Também pode ser compartilhado com outros usuários. Dropbox e Google, ambos oferecem contas gratuitas.

Coletando

07

Verifique se os dados já existem

Vale a pena estender a mão para a comunidade de gerenciamento de informações e o governo da região para verificar se as informações ainda não foram coletadas por outra pessoa. Não há necessidade de duplicar trabalho que já foi realizado.

08

Saiba como seus dados responderão às suas perguntas

Foram realizadas muitas avaliações em que a maioria dos os dados coletados nunca foram utilizados. Ao decidir como você vai usar seus dados antes de começar, você pode reduzir a quantidade de perguntas redundantes e verifique se as perguntas certas são perguntadas.

09

Crie seu modelo de dados antes de coletá-lo

Ao projetar seu modelo de dados antes de coletar dados, ele pode orientar o projeto de pesquisas e a coleta de dados de várias pessoas. Se você deseja conhecer os danos em cada área, em vez de fazer a cada um uma pergunta em aberto, fornecer um modelo garante que os dados sejam coletados de forma consistente.

10

Usar coleta de dados móveis

Em vez de realizar pesquisas em papel, a coleta de dados pode ser concluída usando dispositivos móveis, incluindo telefones e tablets. Isso permite a validação dos dados na entrada, garantindo que os dados coletados sejam de natureza consistente.

Pressione para baixo para mais

10

Existem várias ferramentas disponíveis para ajudar na coleta de dados móveis, algumas gratuitas e outras pagas.

Open Data Kit - Um aplicativo de coleta de dados móvel gratuito para usar.

11

Nomenclatura consistente de variáveis

Se você realizar mais de uma rodada de pesquisas, verifique se os nomes das variáveis usadas nos dados são consistentes com as pesquisas anteriores. Caso contrário, pode ser muito difícil para a pessoa que está analisando os dados corresponder rapidamente às perguntas.

12

Use revisões de dados secundários

Os dados que você deseja, podem existir apenas não em uma única fonte limpa. Considere se é possível reunir esses dados analisando e agrupando outras fontes de dados.

Um exemplo disso é a Cruz Vermelha criando um conjunto de dados dos centros de tratamento de Ebola de diferentes artigos de notícias que os mencionaram para criar um único conjunto de dados consolidado que ainda não existia.

13

Verificar os metadados dos dados

Se você estiver usando ou se referindo a outros dados, verifique os metadados sobre como, quando, onde e por quem os dados foram coletados. Isso fornecerá uma indicação de quão confiáveis são os dados e de que maneira eles podem ser usados.

14

Entenda a diferença entre dados quantitativos e dados qualitativos.

Os dados qualitativos referem-se a histórias que são intencionalmente coletadas e sistematicamente amostradas, compartilhadas, analisadas e analisadas. Isso ajudará você a entender o contexto, produzir insights e pensar sobre quais decisões devem ser tomadas.

Dados quantitativos referem-se a informações que podem ser medidas e anotadas com números, padronizados e normalizados. Ele permitirá que você use métricas para tomar boas decisões.

Planilhas

15

Apenas uma informação por célula

Ter mais de um dado em uma única célula da planilha impede a capacidade de filtrar, somar ou processar um componente da lista de qualquer maneira. Uma maneira de resolver isso é duplicar a linha para cada componente da lista.

16

Aprenda VLookups

Frequentemente, você precisará procurar outros dados com base em um valor no seu conjunto de dados. Um exemplo é que você pode ter os nomes dos distritos em que está trabalhando, mas também deseja os códigos oficiais para eles. Um vlookup pode acessar esses códigos oficiais automaticamente.

Aprenda Tabelas dinâmicas

As tabelas dinâmicas são uma ferramenta extremamente poderosa em planilhas. Eles são usados para resumir, analisar e manipular rapidamente conjuntos de dados.

19

Registre dados em um nível granular e use tabelas dinâmicas para agregar e criar relatórios

Muitas vezes as pessoas organizam seus dados no formulário em relato. No entanto, a agregação restringe possíveis formas de os dados ser organizado no futuro se aparecer uma nova necessidade. É sempre melhor registrar dados da maneira mais básica possível.

20

Não mesclar células em uma planilha

Isso pode quebrar algumas funcionalidades das planilhas como tabelas dinâmicas e filtros. Uma alternativa O método é repetir o valor da célula ou usar cores para indica o intervalo.

21

Mantenha os tipos de dados consistentes nas colunas

Tente garantir que as colunas não contenham misturas tipos de dados como números e palavras.

Ao garantir que todos os tipos de dados sejam iguais, significa que as operações de dados (por exemplo, soma) podem ser realizadas sobre o colunas.

22

Mantenha os nomes dos dados consistentes

A nomeação consistente de dados significa que quando um a planilha é visualizada ou resumida que todas as instâncias corretas dos dados são selecionadas acima. Abaixo, temos a mesma organização escrita de maneiras diferentes e um filtro simples pode não atender a todas.

23

Mantenha todos os dados semelhantes em uma folha ou conjunto de dados

Ao colocar os dados em folhas separadas, dificulta o processo de agregação. Abaixo está uma planilha com uma guia para cada distrito. Poderia ter sido colocado em uma tabela com uma coluna para o distrito, como na coluna A na segunda imagem. Se um relatório for necessário para um distrito, uma tabela dinâmica pode ser usada para gerar isso.

24

Não use cores para representar dados

Quando uma máquina lê os dados ou é copiada para outra folha, há uma chance de que o a formatação é perdida junto com os dados que representa. Crie uma coluna extra para o atributo de dados. Ainda é possível usar cores para realçar os dados.

Gerenciamento de dados

25

Verifique seus dados no local

Uma verificação no local é um processo rápido em que você escolha de 5 a 10 linhas aleatórias dos seus dados e verifique-os cuidadosamente. É especialmente importante fazer isso com dados derivados.

Isso ajuda a verificar se o seu processo está funcionando de forma consistente em todo o conjunto de dados.

26

Verificar máximos, mínimos e somas

Ao revisar seus dados, sempre realize verificações ao extremo. Se você tem alguém com mais de 200 anos em seus dados, você provavelmente pode assumir com segurança que algo precisa ser corrigido no processo.

27

Verificação do sentido

Seus dados fazem sentido? Se você tem mais pessoas afetadas do que a população da área, algo provavelmente deu errado no processo.

28

Verifique se há duplicatas

Para localizar e remover rapidamente duplicatas dos seus dados, use a função de classificação em várias colunas. Isso ordenará linhas duplicadas próximas uma da outra.

29

Use códigos oficiais para representar regiões e distritos sempre que possível

Em muitos países onde operações humanitárias estão sendo realizadas haverá códigos oficiais das áreas de administração. Como os nomes dos lugares podem ser escrito de várias maneiras, especialmente quando caracteres estrangeiros usados, pode ser difícil comparar e juntar conjuntos de dados sem esses códigos. Muitas horas podem ser salvas usando-as.

30

Verificar os relacionamentos

Se você espera certas relações entre dados como um código de administração pertence apenas a um lugar também pode ser verificado com tabelas dinâmicas. Soltando os dois campos nas colunas da linha, mostra todas as correspondências entre os dois colunas de dados. Aqui podemos ver 524 2 05 25 foi usado duas vezes por acidente.

31

Obter coordenadas para dados de pontos

Se você tiver dados que representam um ponto como um hospital ou uma casa em vez de apenas dar um nome ou dar seu endereço, você poderia também coletar coordenadas para ele. Há muitos maneiras possíveis de fazer isso.

Pressione para baixo para mais

31

Um Dispositivo de GPS pode ser usado para obter pontos de coordenadas. A maioria dos smartphones agora contém a tecnologia para fazer isso. Os aplicativos de coleta de dados móveis também podem ajudar no processo.

Os endereços podem ser geocodificado para obter coordenadas. No entanto, em muitos países em desenvolvimento, os endereços não estão bem documentados ou são inexistentes. É melhor registrar os dados de coordenadas ao mesmo tempo que a coleta de dados.

32

HXLize seus dados

HXL é um tipo diferente de padrão de dados, projetado para melhorar o compartilhamento de informações durante uma crise humanitária sem adicionar encargos extras para relatórios. Ao adicionar HXL aos seus dados, você ajuda a tornar esses dados mais compreensíveis e fáceis de processar no futuro.

Visualizando

33

Remover ruído visual

A remoção de elementos visuais desnecessários permite uma interpretação mais fácil dos dados.

34

Alinhamento

Desalinhar elementos pode fazer com que um produto pareça perturbador e difícil para o usuário seguir. Ao alinhar elementos a um sistema de grade, você torna o produto mais atraente e fácil de compreender.

35

Usar iconografia padrão

Ao produzir relatórios, mapas ou painéis usando a iconografia padrão, pode ajudar o espectador a interpretar rapidamente os temas e significados de imagens familiares.

36

Esteja ciente das conotações de cores

As cores têm um significado diferente associado a elas. Se você colorir as áreas de um mapa em vermelho, algumas pessoas assumirão que isso pode indicar uma área de perigo quando essa não for sua intenção. Vale a pena estar ciente de que cores diferentes podem ter significados diferentes em várias culturas.

37

Escolha esquemas de cores legíveis

Cuidado ao escolher cores, especialmente em mapas, para garantir que todos possam lê-las. Seu produto pode ser impresso em preto e branco ou lido por alguém com daltonismo. A escolha de uma escala de cores simples com variação da luminosidade pode resolver esse problema.

38

Faça mapas

Se seus dados tiverem um elemento geográfico, será melhor representá-los como um mapa, em vez de uma tabela de dados. Existem alguns softwares e serviços gratuitos e pagos que podem fazer isso.

Pressione para baixo para mais

38

Faça mapas

QGIS é um software de código aberto usado para criar mapas.

QGIS tutorials é um site que contém muito material de aprendizado para fazer mapas com o QGIS.

39

Faça infográficos e painéis

O relatório da tabela dinâmica pode dizer tudo o que um gerente ou o público precisa saber, no entanto, o engajamento será baixo. Ao criar infográficos e painéis, ajudará a ler os índices.

40

Faça mapas web

Os mapas da Web podem permitir que o usuário interaja com o mapa, ampliando, movimentando, clicando e ativando e desativando as camadas. Esse controle extra significa que mais dados podem ser fornecidos ao usuário e eles podem personalizar o que estão vendo para suas próprias análises.

40

Para quem não sabe codificar, existem serviços por aí que fornecem mapas da web. CartoDB é um serviço popular para colocar mapas da web de seus dados em funcionamento rapidamente.

Para os mais aventureiros, é possível codificar seus próprios mapas da web com JavaScript. Leaflet é uma boa biblioteca para começar.

41

Crie painéis interativos

Os painéis interativos são mais difíceis de criar, mas podem permitir que o usuário faça drill, down, corte e analise realmente um conjunto de dados.

41

Um software pago popular para fazer isso é Tableau. Eles também têm uma versão gratuita para uso pessoal para experimentar.

Se você está familiarizado com a codificação, existem ótimas bibliotecas javascript para criar painéis online. Vale a pena ler em volta d3.js, crossfilter.js and dc.js.

Compartilhamento

42

Compartilhe seus dados em uma planilha e não em PDF

Se você deseja compartilhar sua dados para outras pessoas usarem, em seguida, uma planilha é melhor que uma tabela em um PDF. Tabelas de dados em PDFs nem sempre copiam bem para uma planilha e horas podem ser desperdiçadas colocando os dados em um formato. Considere também compartilhar os dados por trás de mapas e relatórios para outras pessoas fazerem uso.

43

Compartilhe dados abertos no HDX

O intercâmbio de dados humanitários é uma plataforma de compartilhamento de dados para dados humanitários. Ao compartilhar dados, permitimos que mais pessoas façam decisões orientadas por dados informados.

43

Compartilhe relatórios e painéis na Web de Socorro

Ao compartilhar seus produtos na Web de ajuda, outras pessoas da comunidade humanitária podem encontrá-los e usá-los.

Relief Web

44

Use endereços da Web consistentes para atualizações

Um usuário pode marcar ou vincular diretamente ao seu conjunto de dados de outro local. Se você não usar o mesmo endereço da web para obter informações atualizadas, isso vai fazer falta.

45

Incorporar metadados no documento

Os dados geralmente são compartilhados em ad-hoc maneiras através de e-mails e serviços online. Se o meta dados não estão no documento de dados, os dois podem se separar.

Soft skills

46

Comunicação verbal

Enquanto o gerenciamento de informações produtos podem comunicar os dados e uma mensagem, uma informação de gerente geralmente se encontra em reuniões e apresentações. Ser capaz de comunicar verbalmente os dados e simplificar idéias complexas é uma ferramenta poderosa.

47

Ouça

Ouça e seja paciente enquanto entende as necessidades dos usuários de seus produtos de IM. Descubra suas restrições e recursos. Resuma o que você acha que eles querem e faça com que eles concordem. Não construa o que você acha que eles precisam, faça o que eles realmente precisam. Siga as etapas na roda de escuta abaixo para ajudar.

48

Destile as perguntas

A pergunta que o usuário pode colocar originalmente ou o produto que ele pode solicitar nem sempre pode ser o que ele realmente deseja. Às vezes, as pessoas fazem sua própria interpretação de resolver um problema. Ao discutir o problema e as idéias com eles, melhores soluções podem ser encontradas.

49

Não faça suposições sobre o usuário

Ao conversar com usuários ou produzir produtos para o usuário, não faça suposições sobre o que eles já sabem. Ouça suas restrições e conhecimentos.

50

Faça pausas e durma bem

A resposta humanitária pode ser um trabalho estressante e desgastante. Faça intervalos regulares e durma o suficiente!
